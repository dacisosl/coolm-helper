# 쿨메신저 일정 도우미 (coolm-helper)

쿨메신저로 받은 업무 메시지에서 일정·할일을 **버튼 한 번**으로 뽑아 등록하는 교사용 로컬 도구.

> 캡처도, 복붙도, 백그라운드 감시도 없이 — 필요할 때만 꺼내 쓰는 반자동 방식.
> 설치파일 하나로 배포하는 것이 최종 목표입니다. (파이썬 설치 불필요)

## 한눈에 보는 작동 흐름

```
        내 PC 안 (오프라인)                          │  인터넷 (선택)
 ─────────────────────────────────────────────────  │ ──────────────────
                                                    │
  [일정 등록] 버튼 클릭                              │
        │                                           │
        ▼                                           │
  ① 쿨메신저 메시지함(SQLite DB) 복사본 읽기          │
        │      · 원본은 절대 건드리지 않음            │
        ▼                                           │
  ② 날짜·시간·제목 파싱 (정규식 규칙)                 │
        │      · "7월 21일(화) 14:00" → 일정 후보    │
        ▼                                           │
  ③ 개인정보 탐지 → 빨간 글씨 표시                    │
        │      · 전화번호/이름(호칭·명단 대조)         │
        │      · 마스킹하지 않음 — 표시만 하고         │
        │        지울지는 사용자가 편집으로 결정        │
        ▼                                           │
  ④ 미리보기 카드                                    │
        │      · "[학생] 학폭위 심의 3/21 14:00"     │
        │      · 인라인 편집, 눈으로 최종 확인         │
        ▼                                           │
  ⑤ [등록] → 내장 캘린더뷰 + 할일 목록에 저장          │
        │      · 여기까지는 100% 로컬                 │
        └─ (선택) 구글 캘린더에도 올리기 ──────────────┼──▶ 마스킹된 제목만 전송
                                                    │
```

핵심: **⑤까지는 인터넷이 전혀 필요 없습니다.** 구글 캘린더는 원하는 사람만 켜는 부가 기능이고, 켜더라도 온라인으로 나가는 것은 사용자가 미리보기에서 확인한 마스킹된 제목과 일시뿐입니다.

## 개인정보 보호 원칙

- **메시지 원문은 절대 PC 밖으로 나가지 않습니다.** 파싱과 개인정보 탐지는 전부 로컬 규칙(정규식 + 명단 사전)으로 처리합니다.
- 온라인(구글 캘린더)에는 **사용자가 확인·편집한 제목만**, 그것도 직접 [등록]을 눌렀을 때만 올라갑니다.
- 탐지 규칙: 전화번호 / 주민등록번호 / 이름(호칭 패턴 + 학생 명단 `students.txt` + 쿨메신저 교직원 명단 대조)
- 탐지된 부분은 **자동으로 지우지 않고 빨간 글씨로 표시만** 합니다. 지울지 여부는 미리보기에서 사용자가 인라인 편집으로 결정합니다. **사람 눈이 마지막 방어선입니다.**

자세한 구분 원리는 [PRD.md](PRD.md)의 "개인정보 구분 아키텍처" 참고.

## 주요 기능 (우선순위 순)

| 순위 | 기능 | 상태 |
|---|---|---|
| P1 | 일정 등록 (파싱 → 마스킹 → 미리보기 → 저장) | ✅ 구현됨 |
| P2 | **내장 캘린더뷰** — 날짜별 개수 배지, 중요도 칩, 아코디언 상세 편집 | ✅ 구현됨 |
| P2.5 | 설정 모달 (계정/데이터/개인정보/위젯) + 저장 모드 선택(로컬 기본/구글 옵트인) | ✅ 구현됨 |
| P3 | 구글 캘린더 연동 (선택 기능, 설정에서 켬 — 모바일 확인용) | ✅ 구현됨 (설정 필요: calendar_sync/SETUP.md) |
| — | 자동 업데이트 (새 버전 안내 → 자동 설치·재시작) | ✅ 운영 중 (GitHub Releases) |
| — | 데모 모드 — 쿨메신저 없는 PC에서 가짜 쪽지로 체험, 데모 일정 일괄 삭제 | ✅ 구현됨 (설정 → 데이터) |
| P4 | 안내문구 보정 (클라우드 AI, 공개용 글 전용 — 메시지 파이프라인과 코드 분리) | 예정 |

## 실행 / 빌드

```
# 개발 실행
pip install -r requirements.txt
python main.py

# 파서만 콘솔에서 확인 (마스킹된 출력)
python cli_demo.py 7

# 테스트
python -m unittest discover -s tests

# exe 빌드 → dist\CoolmHelper\CoolmHelper.exe
build.bat
# 설치파일: Inno Setup으로 installer.iss 컴파일 → CoolmHelper-Setup.exe
```

## 폴더 구조

```
coolm-helper/
├── README.md            # 이 문서
├── PRD.md               # 상세 기획·아키텍처 문서
├── config.json          # 메시지함 경로, 테이블·컬럼 매핑 (M0에서 생성됨)
├── students.txt         # 학생 명단 (로컬 전용, git 제외)
├── parser/              # DB 리더, 날짜·시간 파싱, 개인정보 탐지 (전부 로컬)
├── ui/                  # 플로팅 위젯, 미리보기 카드, 내장 캘린더뷰
├── store/               # 로컬 저장 (일정·할일 JSON)
└── calendar_sync/       # 구글 캘린더 연동 (선택 기능, 별도 모듈로 격리)
```

## 기술 스택

- Python + PyQt — 플로팅 위젯 UI, 내장 캘린더뷰
- `sqlite3` (표준 라이브러리) — 쿨메신저 DB 읽기, 추가 설치 없음
- 정규식 기반 한국어 날짜/시간 파서 (자체 구현)
- 로컬 JSON 저장 (일정, 할일, 설정)
- Google Calendar API (OAuth) — 선택 기능
- 배포: PyInstaller(exe) + Inno Setup(설치파일)

## 라이선스 / 배포

현재는 개인용. 동료 교사 배포 시:
- 설치파일 하나로 배포 (파이썬 불필요), 첫 실행 시 메시지함 경로 자동 탐지
- 구글 연동은 선택 기능이므로 **구글 설정 없이도 설치 직후 바로 사용 가능**
- 개인정보 처리 고지 문서 동봉 예정
