# 구글 캘린더 연동 설정 (선택 기능)

이 기능을 켜지 않아도 프로그램의 모든 기본 기능(일정 등록, 내장 캘린더, 할일)은
완전하게 동작합니다. 구글 캘린더에도 같이 올리고 싶은 경우에만 아래를 진행하세요.

## 온라인으로 나가는 데이터

구글에 전송되는 것은 **사용자가 미리보기에서 확인한 일정 제목과 시작·종료 일시**
딱 세 가지입니다. 쪽지 원문·발신자·첨부파일명은 전송되지 않습니다
(코드 구조상 전송할 방법 자체가 없습니다 — `google_sync.py` 상단 주석 참고).

## 설정 순서

1. **파이썬 라이브러리 설치**
   ```
   pip install google-api-python-client google-auth-oauthlib
   ```

2. **구글 OAuth 클라이언트 만들기** (최초 1회, 약 10분)
   1. https://console.cloud.google.com 접속 → 새 프로젝트 생성
   2. "API 및 서비스" → 라이브러리 → **Google Calendar API** 사용 설정
   3. "사용자 인증 정보" → 사용자 인증 정보 만들기 → **OAuth 클라이언트 ID**
      - 애플리케이션 유형: **데스크톱 앱**
   4. OAuth 동의 화면에서 게시 상태가 "테스트"라면, 테스트 사용자에
      본인 구글 계정을 추가
   5. 만들어진 클라이언트의 JSON을 다운로드 → 이 폴더에
      `credentials.json` 이름으로 저장

3. **기능 켜기** — 프로젝트 루트 `config.json`에서:
   ```json
   "google_sync_enabled": true
   ```

4. 프로그램 재시작 → 일정 등록 카드에 **[구글에도 등록]** 체크박스가 나타납니다.
   처음 체크해서 등록하면 브라우저가 열리며 구글 로그인을 요청합니다.
   로그인 토큰(`token.json`)은 이 폴더에만 저장되며 외부로 전송되지 않습니다.

## 여러 사람에게 배포할 때

- 구글 정책상 미검증 앱은 "확인되지 않은 앱" 경고가 뜹니다. 개인·소규모 사용은
  테스트 사용자 등록으로 충분하고, 넓게 배포하려면 앱 검증을 신청해야 합니다.
- 그래서 이 프로그램은 구글 연동 없이도 완전히 쓸 수 있게 설계돼 있습니다.
  동료에게는 기본(로컬) 모드로 배포하고, 원하는 분만 이 문서로 개별 설정하세요.

## 끄기 / 초기화

- 끄기: `config.json`의 `google_sync_enabled`를 `false`로
- 계정 연결 해제: 이 폴더의 `token.json` 삭제
